# 剧场假发勾织维修单

## 启动

```bash
npm install
npm start
```

默认地址：http://localhost:3909

数据保存在`data/db.json`，后续可以继续增量迭代。

## 模块说明

### 1. 看板
展示系统概览统计和演出前优先清单，快速了解假发档案总数、可演出数量、演出排期数量、维修单数量和紧急维修数量。

### 2. 演出排期（新增）

用于登记剧目演出的排期信息，关联需要使用的假发。

**功能特性：**
- 登记剧目、场次日期、角色信息
- 从已有假发档案中选择需要使用的假发
- 排期列表实时显示关联假发的当前可用状态（可演出/需要维修/紧急维修/已归还入库）
- 支持按剧目、角色搜索排期
- 支持按排期状态（已排期/待确认）筛选
- 提供"确认排期"快捷操作按钮

**排期状态说明：**
- **已排期**：排期已确认，正常安排中
- **待确认**：排期信息待核实确认

**假发状态实时显示：**
- 🟢 可演出：假发状态良好，可正常使用
- 🟡 需要维修：假发有小问题，需要维修后使用
- 🔴 紧急维修：假发问题严重，需紧急维修
- ⚫ 已归还入库：假发已归还入库

### 3. 假发档案
管理所有假发的基础信息，包括角色、剧目、发色、发网尺寸、发际线类型、存放位置、演出日期、可用状态和备注。

### 4. 维修流转
登记和跟踪假发的维修任务，包括维修类型、处理人、截止日期、状态、处理内容和结果。完成维修后会自动将关联假发标记为"可演出"状态。

### 5. 耗材台账（新增）

管理维修所需的各类耗材（勾针、蕾丝、发束、定型用品等），记录库存数量和安全库存阈值，低于安全库存的项目会自动高亮提醒。

**功能特性：**
- 登记耗材名称、库存数量、安全库存和备注
- 自动计算库存状态（库存充足 / 库存不足 / 库存告警）
- 低于安全库存的耗材卡片会以红色边框和背景高亮显示
- 库存告警的耗材自动排在列表最前面，方便优先补货
- 支持按耗材名称、备注搜索
- 看板统计显示耗材总种类数和库存告警数

**库存状态说明：**
- 🟢 **库存充足**：库存数量 ≥ 安全库存
- 🟡 **库存不足**：库存数量 < 安全库存，但 ≥ 安全库存的 50%
- 🔴 **库存告警**：库存数量 < 安全库存的 50%，或库存为 0

**注意**：维修单登记时暂不自动扣减库存，需手动更新耗材台账的库存数量。

## 启动后验证方式

1. **启动服务**：执行 `npm start`，访问 http://localhost:3909
2. **查看看板**：确认顶部统计卡片中显示"耗材种类"和"库存告警"两项
3. **进入耗材台账**：点击顶部"耗材台账"标签页
4. **验证列表**：
   - 确认显示 5 条示例耗材数据
   - 确认"蕾丝前额"（库存 3 < 安全库存 5）和"定型发胶"（库存 2 < 安全库存 4）显示为红色高亮的库存告警状态
   - 确认库存告警的耗材排在列表最前面
5. **新增耗材**：
   - 填写耗材名称、库存数量、安全库存等信息
   - 点击"保存耗材"按钮
   - 确认新耗材出现在列表中
6. **搜索功能**：在搜索框输入"发胶"或"蕾丝"，确认能正确过滤结果
7. **库存状态验证**：
   - 库存 ≥ 安全库存 → 绿色"库存充足"标签
   - 库存 < 安全库存但 ≥ 50% → 橙色"库存不足"标签
   - 库存 < 安全库存 50% 或为 0 → 红色"库存告警"标签 + 卡片高亮
